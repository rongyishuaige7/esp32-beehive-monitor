# 安全、网络、数据与不适用场景

## 严格不适用场景

- 本项目不是蜂群健康诊断、疾病识别、动物健康、养蜂生产决策、气象预报、烟雾/燃气/火灾报警、环境安全、应急响应或无人值守系统。
- 不得依据温湿度、光照、气压、声音幅度、MQ-2 原始 ADC、固定阈值标签、LED、串口、HTTP JSON、App 页面、CI 或构建结果作出蜂群状态、蜂箱状态、火灾/燃气/烟雾、天气、安全、告警送达或有人处理的结论。
- 不得将本项目用于火源、燃气、市电、危险场所、动物健康、生产控制、人身安全、应急响应或任何需要可靠报警/人工处置的场景。

## 凭据、网络与数据

- `firmware/src/wifi_credentials.h` 只能保存在本机，已被 Git 忽略。绝不提交、截图、录屏、粘贴到 Issue 或写入日志。
- App 保存的目标地址、串口输出、网络日志、实物照片和演示视频都可能泄露私网信息。请勿公开 SSID、密码、私网 IP/MAC、位置、蜂场资料、传感器记录、EXIF/GPS 或可识别个人的数据。
- 有凭据后，固件只提供无 TLS、无认证、无访问控制、无审计、无设备身份和无速率限制的明文 HTTP；只可用于短期、隔离、可信的教学局域网。禁止公网暴露、端口转发、公共 Wi-Fi、共享热点或不可信网络。
- App 的一次“本次请求成功”只表示当次 HTTP 请求收到了可解析的本地 JSON；不表示设备在线、持续连接、服务健康、数据准确或有人处理。

## 硬件边界

- ESP32 GPIO 与 ADC 输入不得超过 3.3 V。MQ-2、LM386 或其他模块的 AO 输出必须先按实际模块完成电平调理。
- 所有模块都必须确认额定电压、电平、供电、电流、限流、上拉、共地和实际接线；不要让 GPIO 直接驱动大电流负载。
- 未完成当前 commit 的实物复测前，不要把源码引脚、阈值、LED 或 HTTP 行为写成已验证接线、可靠网络或安全功能。

## 报告问题

请勿在公开 Issue 中提交真实凭据、私网信息、位置、传感器记录、照片元数据或可识别个人的数据。仓库没有承诺私密报告入口；请只提交最小化、已脱敏的复现材料。
